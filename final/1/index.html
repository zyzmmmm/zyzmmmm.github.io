1. Randomly Sampled Texture

Create an output image by randomly sampling square patches from the input texture sample. The patches are tiled until the output size is achieved. Black borders may remain if patches do not fit exactly.

Parameters: Patch size, Output size

2. Overlapping Patches

Generate an output image using overlapping patches. Compute the SSD (Sum of Squared Differences) for each patch and choose the one with the lowest cost below a given tolerance.

Parameters: Patch size, Overlap size, Tolerance

3. Seam Finding

Remove edge artifacts by computing a seam along pixels with the minimum SSD cost. Masks are used to blend overlapping patches seamlessly.

Illustrations:

  • (b) Pixelwise SSD cost

4. Additional Quilting Results

Apply the quilting algorithm to your own input textures to generate outputs larger than the input texture size. Include both input and output images.

5. Texture Transfer

Generate an output image where the texture from a sample is transferred onto a target image while preserving its structure. This is achieved by adding a cost term for texture similarity to the target image during the quilting process.

Parameters: Sample image, Target image

6. Quality of Results / Report

Evaluation based on the quality of results and the clarity of the report. (0=poor, 5=average, 10=great)
